# Product Brief: {Nombre del Proyecto}

## 1. Problema de negocio

Describir el problema que se quiere resolver. Responder:
- ¿A quien afecta este problema?
- ¿Cual es el impacto actual (costos, ineficiencia, riesgo)?
- ¿Por que es necesario resolverlo ahora?

## 2. Solucion propuesta

Describir la solucion a alto nivel. Responder:
- ¿Que producto/sistema se va a construir?
- ¿Como resuelve el problema descrito?
- ¿Cual es el diferenciador frente a alternativas existentes?

## 3. Propuesta de valor

Explicar el valor que entrega el producto. Responder:
- ¿Que beneficios concretos obtienen los usuarios/stakeholders?
- ¿Cual es el retorno esperado (ROI, eficiencia, ventaja competitiva)?

## 4. Alcance (in / out)

### Dentro del alcance
- [ ] Funcionalidad A
- [ ] Funcionalidad B

### Fuera del alcance (explicitamente)
- Funcionalidad X — motivo: fase 2
- Funcionalidad Y — motivo: otro sistema

## 5. Restricciones y supuestos

### Restricciones
- Presupuesto maximo: ${monto}
- Tiempo limite: {fecha}
- Tecnologia obligatoria: {stack}
- Compliance: {normativa}

### Supuestos
- El equipo tendra acceso a {recurso}
- La API externa {nombre} estara disponible
- El cliente proveera {dato/recurso}
